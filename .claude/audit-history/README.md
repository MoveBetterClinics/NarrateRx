# Audit history

Generated by the `/audit` slash command (see `.claude/commands/audit.md`).

## Files

- `<YYYY-MM-DD-HHMM>.md` — one report per audit run, written automatically. Contains the prioritized punch list (P0/P1/P2) plus the raw output of each agent for traceability.
- `.last-audit` — single line containing the SHA of HEAD at the time of the last successful audit. The next `/audit` (default mode) scopes `bug-hunter` and `tenant-isolation-auditor` to commits in `<.last-audit>..HEAD`. The `ui-reviewer` always does a full sweep (visual drift is cumulative).

## Resetting the baseline

To force the next `/audit` to start from a clean baseline:

```bash
cd "/Users/qbook/Claude Projects/NarrateRx" && git rev-parse HEAD > .claude/audit-history/.last-audit
```

## Manual full sweep

```bash
/audit full
```

Same agents, no scoping. Higher cost, but catches anything previous since-last passes missed.
